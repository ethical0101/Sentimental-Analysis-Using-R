# PROJECT OVERVIEW & METHODOLOGY DOCUMENT
# AI-Based Multi-Dimensional Sentiment & Behavioral Intelligence System

## EXECUTIVE SUMMARY

This project presents an advanced sentiment analysis system that transcends traditional 
polarity classification by incorporating multi-dimensional analysis including emotion 
detection, sarcasm identification, topic modeling, temporal trend analysis, and machine 
learning classification comparison. Built entirely in R, the system provides both 
command-line and interactive dashboard interfaces for comprehensive text analytics.

---

## 1. INTRODUCTION

### 1.1 Background

Sentiment analysis has become a crucial tool for understanding public opinion, customer 
feedback, and social media discourse. Traditional sentiment analysis focuses solely on 
positive/negative classification, missing nuanced emotional states, sarcasm, and 
behavioral patterns.

### 1.2 Motivation

Modern text analysis requires:
- Multi-dimensional emotional understanding
- Detection of linguistic subtleties (sarcasm, irony)
- Identification of thematic patterns (topics)
- Temporal trend analysis
- Robust machine learning validation
- User-friendly interfaces for non-technical users

### 1.3 Objectives

This system aims to:
1. Perform lexicon-based sentiment analysis with intensity scoring
2. Detect 8 distinct emotions using NRC lexicon
3. Identify sarcastic statements through rule-based patterns
4. Discover latent topics using LDA
5. Analyze sentiment trends over time
6. Compare ML algorithms (Naive Bayes, SVM) with lexicon methods
7. Provide comprehensive visualizations
8. Deliver an interactive web dashboard
9. Export analysis results for further research

## 3. LITERATURE REVIEW

### 3.1 Sentiment Analysis Approaches

**Lexicon-Based Methods:**
- Utilize predefined sentiment dictionaries
- Fast and interpretable
- Limited by lexicon coverage
- Example: Bing Liu Lexicon, AFINN

**Machine Learning Methods:**
- Learn from labeled data
- Better generalization
- Require training data
- Examples: Naive Bayes, SVM, Neural Networks

**Hybrid Approaches:**
- Combine lexicon and ML
- Leverage strengths of both
- Our system implements this approach

### 3.2 Emotion Detection

**NRC Emotion Lexicon (Mohammad & Turney, 2013):**
- 8 basic emotions: anger, anticipation, disgust, fear, joy, sadness, surprise, trust
- Crowd-sourced word-emotion associations
- Widely used in academic research
- Implemented in syuzhet R package

### 3.3 Sarcasm Detection

**Approaches:**
- Rule-based: Pattern matching (our approach)
- Feature-based ML: Punctuation, emoji, linguistic features
- Context-based: Sequential models (LSTM, BERT)
- Hybrid: Combination of above

**Our Implementation:**
- Excessive punctuation detection
- Emoji/expression analysis
- Quotation mark usage
- Contradiction detection
- Confidence scoring

### 3.4 Topic Modeling

**LDA (Latent Dirichlet Allocation):**
- Probabilistic generative model
- Discovers latent topics in document collections
- Represents documents as topic distributions
- Blei et al., 2003

**Applications:**
- Theme discovery
- Document clustering
- Content recommendation

### 3.5 Time-Series Sentiment Analysis

**Temporal Analysis:**
- Sentiment trend detection
- Peak/valley identification
- Moving averages
- Seasonality patterns

**Applications:**
- Brand monitoring
- Crisis detection
- Campaign effectiveness

### 4.2 Algorithm Details

#### 4.2.1 Sentiment Analysis Algorithm

**Input:** Text document d
**Output:** Sentiment score s, polarity p, intensity i

```
Algorithm: LEXICON_SENTIMENT_ANALYSIS(d)
1. Tokenize d into words W = {w₁, w₂, ..., wₙ}
2. Load positive lexicon P and negative lexicon N
3. Initialize pos_count = 0, neg_count = 0
4. For each word wᵢ in W:
     If wᵢ ∈ P: pos_count++
     If wᵢ ∈ N: neg_count++
5. Detect intensifiers I and negations G
6. Calculate multiplier m = 1 + (|I| × 0.5)
7. Calculate base score: s_base = (pos_count - neg_count) × m
8. If |G| is odd: s = -s_base
   Else: s = s_base
9. Classify polarity:
     If s > 0: p = "Positive"
     If s < 0: p = "Negative"
     If s = 0: p = "Neutral"
10. Compute intensity:
     If s > 5: i = "Strong Positive"
     If 1 ≤ s ≤ 5: i = "Mild Positive"
     If s = 0: i = "Neutral"
     If -5 ≤ s ≤ -1: i = "Mild Negative"
     If s < -5: i = "Strong Negative"
11. Return (s, p, i)
```

**Time Complexity:** O(n) where n is number of words
**Space Complexity:** O(|P| + |N|) for lexicon storage

#### 4.2.2 Emotion Detection Algorithm

**Input:** Text document d
**Output:** Emotion vector E, dominant emotion e_d

```
Algorithm: EMOTION_DETECTION(d)
1. Tokenize d into words W
2. Initialize emotion scores: E = {anger: 0, anticipation: 0, disgust: 0, 
                                   fear: 0, joy: 0, sadness: 0, 
                                   surprise: 0, trust: 0}
3. For each word wᵢ in W:
     Lookup wᵢ in NRC lexicon
     For each emotion e associated with wᵢ:
       E[e]++
4. Calculate dominant emotion: e_d = argmax(E)
5. If all E[e] = 0: e_d = "Neutral"
6. Calculate intensity: intensity = Σ E[e]
7. Return (E, e_d, intensity)
```

**Time Complexity:** O(n × k) where k = 8 emotions
**Space Complexity:** O(n + |NRC lexicon|)

#### 4.2.3 Sarcasm Detection Algorithm

**Input:** Text d, sentiment score s
**Output:** Sarcasm flag is_sarcasm, confidence c

```
Algorithm: DETECT_SARCASM(d, s)
1. Initialize sarcasm_score = 0
2. Check for indicators:
     excessive_punct = REGEX_MATCH(d, "[!?]{2,}")
     laughter = REGEX_MATCH(d, "😂|lol|haha|lmao")
     quotes = REGEX_MATCH(d, "\"[^\"]*\"")
     caps = REGEX_MATCH(d, "[A-Z]{3,}")
     exaggeration = REGEX_MATCH(d, "best|worst|ever|always|never")
     neg_context = REGEX_MATCH(d, "but|however|unfortunately")
3. If s > 0:  # Positive sentiment
     If excessive_punct: sarcasm_score += 2
     If laughter: sarcasm_score += 2
     If quotes: sarcasm_score += 1
     If caps: sarcasm_score += 1
     If exaggeration AND (excessive_punct OR laughter): sarcasm_score += 2
     If neg_context: sarcasm_score += 2
4. If s = 0 AND quotes: sarcasm_score += 3
5. Calculate confidence: c = (indicator_count / max_indicators) × 100
6. If sarcasm_score ≥ 3:
     is_sarcasm = TRUE
   Else:
     is_sarcasm = FALSE
7. Return (is_sarcasm, c)
```

**Time Complexity:** O(|d|) for regex matching
**Space Complexity:** O(1)

#### 4.2.4 Topic Modeling Algorithm (LDA)

**Input:** Document collection D, number of topics k
**Output:** Topic-word distribution β, document-topic distribution θ

```
Algorithm: LDA_TOPIC_MODELING(D, k)
1. Create Document-Term Matrix (DTM):
     Vocabulary V = unique words in D
     DTM[i,j] = frequency of word j in document i
2. Initialize:
     β = random topic-word distributions (k × |V|)
     θ = random document-topic distributions (|D| × k)
3. Gibbs Sampling (iterations = 1000):
     For each document d in D:
       For each word w in d:
         Sample topic z from P(z|w,d) ∝ β[z,w] × θ[d,z]
         Update β and θ based on assignments
4. Extract top n_terms words per topic from β
5. Assign dominant topic to each document using θ
6. Return (β, θ, top_terms, assignments)
```

**Time Complexity:** O(iterations × |D| × avgDoc × k)
**Space Complexity:** O(|D| × |V| + k × |V|)

#### 4.2.5 Machine Learning Pipeline

**Naive Bayes Classifier:**
```
P(class|features) = P(class) × ∏ P(feature|class) / P(features)

With Laplace smoothing:
P(feature|class) = (count(feature, class) + α) / (count(class) + α|V|)
where α = 1 (smoothing parameter)
```

**SVM Classifier:**
```
Find hyperplane that maximizes margin:
f(x) = w·x + b
where w is weight vector, b is bias

Linear kernel: K(x,x') = x·x'
Optimization: min ½||w||² + C Σ ξᵢ
where C is regularization parameter, ξᵢ are slack variables
```

### 4.4 Evaluation Metrics

**Sentiment Classification:**
- Accuracy = (TP + TN) / (TP + TN + FP + FN)
- Precision = TP / (TP + FP)
- Recall = TP / (TP + FN)
- F1 Score = 2 × (Precision × Recall) / (Precision + Recall)

**Topic Modeling:**
- Perplexity (lower is better)
- Topic Coherence (higher is better)
- Top-N term distinctiveness

**Sarcasm Detection:**
- Confidence score (rule-based)
- Manual validation on sample

## APPENDIX A: GLOSSARY

- **Sentiment Analysis:** Computational identification of opinions in text
- **Polarity:** Classification of text as positive, negative, or neutral
- **Lexicon:** Dictionary of words with sentiment scores
- **NRC Lexicon:** Emotion lexicon with 8 emotion categories
- **LDA:** Latent Dirichlet Allocation, probabilistic topic model
- **TF-IDF:** Term Frequency-Inverse Document Frequency weighting
- **DTM:** Document-Term Matrix, numerical representation of text corpus
- **Confusion Matrix:** Table showing classifier performance
- **Precision:** Proportion of correct positive predictions
- **Recall:** Proportion of actual positives correctly identified
- **F1 Score:** Harmonic mean of precision and recall
